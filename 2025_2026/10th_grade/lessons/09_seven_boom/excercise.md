# משימת בית - משחק "שבע בום"

מטרת המשחק: השחקן אמור להקליד מספר או את המילה boom עבור סדרת מספרים עוקבים על פי הכלל הבא - עבור מספרים שמתחלקים ב7 או מכילים את הספרה 7 השחקן יקליד boom. לכל מספר אחר השחקן אמור להקליד את המספר.

## כללים

- אין צורך לבדוק את תקינות הקלט, אפשר להניח שהמשתמש תמיד מקליד מספר תקין או את המילה boom באותיות קטנות
- שמות של פונקציות ומשתנים 
  - השתמשו בשמות מתאימים לתפקיד
  - ככל שהמרחק בין הגדרת המשתנה לשימוש במשתנה קטן יותר, אפשר להשתמש בשם קצר ופשוט יותר. לדוגמא: אם יש לנו פונקציה שמקבלת משתנה מספרי שלם אחד ומבצעת איתו משהו - n הוא שם מוצלח. לעומת זאת אם אנחנו שומרים את המספר הנוכחי שהמשתמש אמור להכניס במשך כל התכנית - currNum או currentNumber הם בחירה יותר מוצלחת מ n

## שלבי עבודה מומלצים

### בדיקת התחלקות ב7

ממש את הפונקציה isDividableBy7. 

קלט - מספר לבדיקה
פלט - האם המספר מתחלק ב7 

מזכיר לכם: כדי לבדוק פונקציה אחת מומלץ להתחיל מקוד פשוט שמקבל קלט מהמשתמש, שולח אותו לפונקציה ומחזיר את התשובה, באופן כזה תוכלו לבדוק האם הפונקציה שלכם בכל מיני מצבים ולראות שהיא עובדת נכון. 

דוגמא לקוד בדיקה של פונקציה בשם FunctionForTesting

```csharp
static int FunctionForTesting(int number)
{
    // Example logic: double the number
    return number * 2;
}

static void Main()
{
    Console.Write("Enter a number: ");
    int input = int.Parse(Console.ReadLine());

    int result = FunctionForTesting(input);

    Console.WriteLine($"Result: {result}");
}
```

### בדיקת האם 7 מוכל במספר

ממש את הפונקציה isContainSeven

קלט - מספר שלם כלשהו

פלט - האם המספר מכיל את הספרה 7 

גם כאן, מומלץ להשתמש בקוד בדיקה לפני שמשלבים את הפונקציה בכלל התכנית

לא זוכרים איך מוצאים ספרה בתוך מספר גדול, רמז - חזרו לתרגיל של טבלאות מעקב ותראו כיצד אפשר לעבור על מספר שלם ספרה אחר ספרה.

### בדיקה האם המספר הוא "בום" או לא 

ממש את הפונקציה isBoom

קלט - מספר שלם כלשהו

פלט - האם המספר הוא "בום" (מכיל 7 או מתחלק ב7)

רמז - השתמשו בשתי הפונקציות הקודמות

### לולאה מספרית פשוטה

כתבו את הלולאה הראשית, באופן כזה שהיא מצפה מהמשתמש להקליד כל פעם את המספר הבא (יעני שבע בום , בלי החלק של השבע והבום)

### חברו את החלקים

כעת השתמש בפונקציה isBoom בצורה כזו שכאשר המשתמש מקליד את הדבר הנכון (מספר או המילה boom) היא תתקדם למספר הבא, וכאשר הוא טועה היא תדפיס הודעת שגיאה ותעצור. 

הנחיות : יש לטפל בשלושת המקרים הבאים:
- אם המספר הנוכחי הוא "בום"
  - כל דבר שהמשתמש יקליד חוץ מboom הוא טעות
- אם המספר הנוכחי הוא לא "בום" 
  - הקלדת boom היא טעות
  - הקלדת מספר אחר מאשר המספר הבא היא גם טעות

המלצה: השתמש בלולאת while עם דגל (משתנה בוליאני) לעצירה

### הוסיפו ניקוד

כאשר המשתמש נכשל, הדפיסו את הניקוד שלו, זאת אומרת - כמה פעמים הוא ענה נכון

### הוסיפו אקראיות

במקום להתחיל את המספר מ1 תמיד, התחילו ממספר אקראי בין 1 ל 100 

לא זוכרים איך יוצרים מספר אקראי, הסתכלו בדוגמא הבאה. שימו לב שהיצירה של rnd נדרשת רק פעם אחת בכל התכנית.

```csharp
int lower = 1;
int upper = 10; // upper bound is exclusive

Random rnd = new Random();
int value = rnd.Next(lower, upper);

Console.WriteLine(value);
```
